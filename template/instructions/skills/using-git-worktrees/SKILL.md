# Using Git Worktrees

Use this for parallel tasks or when isolating risky changes.

## Procedure

1. Create a dedicated branch/worktree for the task.
2. Verify clean baseline before changes.
3. Keep changes scoped to the worktree purpose.
4. Remove or archive worktree after merge/close.

## Rules

- Name branches clearly by task.
- Keep each worktree focused on one coherent task or change set.
