# Writing Plans

Use this after design approval.

## Procedure

1. Break work into small tasks that can be completed and verified independently.
2. Include exact file targets per task.
3. Define verification commands for each task.
4. Flag dependencies and ordering constraints.

## Plan quality bar

- Tasks are small enough for fast review.
- Verification is concrete and reproducible.
- Scope matches the approved design.
