# Requesting Code Review

Use this when implementation is complete and verified.

## Procedure

1. Summarize problem, approach, and impacted files.
2. List high-risk areas and behavior changes.
3. Include verification evidence.
4. Ask for review focus on correctness, regressions, and security.

## Output

- Change summary
- Risk list
- Verification summary
- Open questions for reviewers
