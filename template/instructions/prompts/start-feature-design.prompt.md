---
name: start-feature-design
description: Run a structured design and planning kickoff for a new feature.
---

Use this workflow to kick off a new feature.

1. Clarify requirements, constraints, and non-goals.
2. Produce an approval-ready design summary.
3. Produce a task plan with file targets and verification commands.
4. Save the design to `docs/design/<feature>.md` and the plan to `docs/plans/<feature>-plan.md`.

If requirements are ambiguous, ask focused questions before finalizing the plan.
